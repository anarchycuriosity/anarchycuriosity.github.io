内存泄漏/溢出与指针置空

2026-02-12
阅读时长: 6 分钟
... 次阅读

背景

最近在复习链表，意识到自己对指针和内存这一部分的了解很肤浅，所以复习。我们在课程中经常会听到，如果为一个指针开辟了一个新的区域但是最后却不delete和置空指针会导致内存泄漏和内存溢出，野指针。很多人包括我机械地背诵这一危害，但是却并不能很清楚这些危害到底指的是什么。

至于为什么会忽视，我想是数组和链表的特点使然。因为数组中并不涉及频繁的插入与删除，而且stl的vector容器也让我们省去了管理内存的麻烦，这让数组阶段的学习中内存的风险很低，但在链表中，这一风险被大大放大了。

指针的本质

指针是一种基本的数据类型,和int一样属于基本类型

它储存的是一个十六进制的编号比如0x7ffeb3

如果把指针扒光,其实它就是一个正整数,但编译器给他打了个标签(类型),并让他拥有两个特性.

  • 解引用(dereference)

对于普通的整数a = 100,无法让他把100号房间的东西拿给我.但对于指针p = 100,可以通过*p瞬间传送到那个内存位置

  • 步长意识(the scale awareness)

指针知道自己指向的东西有多大,比如p1和p2

1
2

char *p1 = (char*)100;
int *p2 = (int*)100;

一个指向char,一个指向int,p1+1和p2+1,一个结果是101,一个是104

内存溢出是结果,内存泄漏是原因之一

void deleteAtIndex(int index)
    {
        if (index < 0 || index >= _size)
        {
            return;
        }
        Linked_node *cur = _dummy_head;
        while (index--)
        {
            cur = cur->next;
        }
        Linked_node *tmp = cur->next;
        cur->next = cur->next->next;
        delete tmp;
        // delete命令指示释放了tmp指针原本所指的那部分内存，
        // 被delete后的指针tmp的值（地址）并非就是NULL，而是随机值。也就是被delete后，
        // 如果不再加上一句tmp=nullptr,tmp会成为乱指的野指针
        // 如果之后的程序不小心使用了tmp，会指向难以预想的内存空间
        tmp = nullptr;
        _size--;
    }

其中有几个关键动作,delete tmp和tmp = null.

引入两个概念,指针变量本身的值(也就是它记录的地址)和指针指向的那块内存区域的状态.

比如a指针和b指针.当我们delete a后,a所在的地址的实际位置可以被别的变量或者程序使用,也就是告诉操作系统"这块地我不用了"

但是a依然拿着这个地址.好巧不巧a被剥夺了实际地址后,b就被划分到了同一块地址.(ps:这里的同一块地址是a被delete前的旧址,注释中的随机值 和垃圾值指的是这个地址已经失去了意义,因为delete后这个地址对应的内存已经没有东西了)

如果这时我们a->val = sth.会导致b的数据被更改.我们依然可以对a解引用,这时a和b的编号是一致的,相当于a和b拿着一样的钥匙,这是很危险的.

所以这时我们需要抹消掉a的值,a = nullptr.(ps:NULL的本质是0的宏,nullptr是专门给指针用的,如果不使用nullptr使用NULL也可以但不是很推荐)

当我们删除掉某个节点,也就是让链表绕过那个节点.此时如果不做delete操作,由于链表和这个删除的元素已经没有关系了,我们将永远找不到这个删除的元素的位置,它就像一个脱缰的野马,占着茅坑不拉屎,也就是内存泄漏,我们的程序有着无法管理的内存.

当内存泄漏到了一定程度的时候,就会导致内存溢出.

只有让指针置空才能保证不会通过这个指针误操作.

一些问题

Q:我已经delete掉了tmp,我后面根本不会使用tmp这个指针名字来操作这个地址了,是不是意味着没有必要置空指针.

A:当项目很大的时候,很难确定你的同事/你自己在日后会不会再次使用这个tmp.这就埋下了风险.

Q:为什么有些地方会置空指针,有些地方只会delete而不会置空.

A:在链表类中，我们往往只会有两个指针出现.一个是傀儡头的指针,还有一个是我们用来遍历的临时指针,往往用cur来命名.因为cur是临时对象,只在我们调用类的函数的时候会出来,在函数结束的时候,也就是函数大括号结束的时候,这个临时对象的生命周期就结束了,所以甚至没有必要delete.而傀儡头的指针,它属于类的一个成员,是本体成员而不是衍生成员.衍生成员所指向的内存并不归这个链表类管理,所以我们需要在析构函数中写明白删除衍生成员指向内存的逻辑.但是本体成员–这个傀儡头指针的值会在这个类结束的时候清空,也就是指针置空(= nullptr).

总结:所有的内存包括临时变量的指针,链表衍生成员指针的内存,链表本体成员指针的内存都需要delete,类的本体成员指针结束的时候只会结束指针的值,所以不需要置空.但临时变量的指针需要置空,它不属于任何类,虽然它的生命将在到达}的时候结束,但是我们为了防止误操作,需要置空,本质是一种防御性编程.

链表析构函数,由于类结束的时候无法管理到傀儡头以外的节点的指针和内存,所以我们需要额外操作.

cur需要不断在循环中更新状态,所以我们不能置空它.当cur的循环结束的时候我们不再需要对cur进行操作了,置空保险.

~MyLinkedList()
{
    Linked_node *cur = _dummy_head;
    while (cur != nullptr)
    {
        // 先把下一站的“地址”抄下来
        Linked_node *next_node = cur->next;
        
        // 拆掉现在的房子（释放内存）
        delete cur;
        
        // 这里的 cur 不需要置空，因为 cur 本身是局部变量，马上就消失了
        // 关键是上面的 delete 已经把内存还给了系统
        
        // 走向下一站
        cur = next_node;
    }
    // 循环结束，所有节点都还给了系统
}
