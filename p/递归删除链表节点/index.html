递归删除链表节点

2026-02-12
阅读时长: 2 分钟
... 次阅读

核心思路

我们使用1-2-3-2来删除2来进行模拟

思路

递归删除的思路有两个核心因素

  • 一个是触底位置，也就是终止条件（如果没有它递归不会停止）

  • 一个是挖掘（递归）的条件，以及当前挖掘层的对应任务。比如，若当前节点的值是目标值（挖掘的条件）则新建一个节点new_head赋值为下挖的返回值，我喜欢把这称之为挖掘的任务。

  • 另一个是返回的任务，由小到大，由深及浅地完成对应任务。是删除节点直接返回深层节点，或是让当前节点接上深层节点并返回。

模拟

可以想象一个小队向下挖东西，如果挖到符合要求的地块，则下派a往下挖，同理a下派b往下挖。当挖到地基的时候（nullptr）则返回，从b-》a进行一个接力。如果b返回的时候a那层地基埋有炸弹，就不走a那层了，直接绕过a（返回的任务）。

// 我们可以使用递归的方式来写删除元素
struct ListNode
{
    int val;
    ListNode *next;
    ListNode() : val(0), next(nullptr) {}
    ListNode(int x) : val(x), next(nullptr) {}
    ListNode(int x, ListNode *next) : val(x), next(next) {}
};

class Solution
{
public:
    ListNode *removeElements(ListNode *head, int val)
    {
        // 基础情况
        // 需要先处理如果是空链表的情况
        // 当到达最后一层的时候，递归结束，然后返回
        if (head == nullptr)
        {
            return nullptr;
        }
        if (head->val == val)
        {
            // 新建节点做递归并删除旧的节点
            // 如果本节点需要被删除，则返回之后的节点
            ListNode *new_head = removeElements(head->next, val);
            delete head;
            return new_head;
        }
        else
        {
            // 如果本节点不需要被删除，则让本节点接上下一层递归的节点然后返回
            head->next = removeElements(head->next, val);
            return head;
        }
    }
};
